Foundry at a glance

A few high-signal points from the project documentation that are worth exposing directly on the Pages landing page.

Markdown-first content

Pages and posts live in files with frontmatter, version cleanly in Git, and can be extended with taxonomies, fields, and language-aware routing.

Theme and plugin system

Foundry keeps rendering isolated in themes and extension logic in plugins, with explicit hook points and generated plugin registration.

Dependency-aware rebuilds

The dependency graph now tracks taxonomy outputs too, which lets incremental rebuild planning target document and archive URLs more precisely.

Admin and preview guardrails

The admin surface now requires admin.access_token, Markdown raw HTML is config-controlled, and the preview/admin HTTP server uses timeouts.
